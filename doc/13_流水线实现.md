## 1. 架构概览

### 1.1 五级流水线

```
       IF           ID           EX           MEM          WB
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
   │ PC → IM │→│ Ctrl    │→│ ALU     │→│ DM      │→│ RF[rd]←│
   │ PC+4    │ │ RF 读   │ │ 转发MUX │ │ 字节/半 │ │ WD MUX  │
   │         │ │ EXT     │ │ 分支决议 │ │ 字处理  │ │         │
   └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘
        │           │           │           │           │
    ┌───▼───┐  ┌───▼───┐  ┌───▼───┐  ┌───▼───┐
    │IF/ID  │  │ID/EX  │  │EX/MEM │  │MEM/WB │
    │寄存器  │  │寄存器  │  │寄存器  │  │寄存器  │
    └───────┘  └───────┘  └───────┘  └───────┘
```

### 1.2 与单周期版本的关键差异

| 特性 | 单周期 | 流水线 |
|------|--------|--------|
| 每指令周期数 | 1 (整个周期) | 5 级 × 1 cycle/级 |
| CPI (理想) | 1.0 | 1.0 (5 条指令同时在不同级) |
| 时钟频率 | 受最慢指令限制 | 受最慢单级限制（更高） |
| 控制信号 | 直接连线 | 随指令逐级传递 |
| 数据冒险 | 不存在 | 需转发 + 阻塞 |
| 控制冒险 | 不存在 | 需冲刷 |
| PC 更新 | 每条指令一次 | 每周期一次（PC+4 默认） |
| RF 写 | posedge 写, 同周期组合读 | posedge 写, 下周期组合读 |

### 1.3 关键设计原则

1. **所有时序逻辑用 `posedge clk`**，流水线寄存器用 `negedge clk`（跟上老师参考模板）
2. **控制信号随指令流动**：在 ID 级译码，逐级传递，不需要的级直接透传
3. **前递优于阻塞**：能转发就转发，只在 Load-Use 时插入气泡
4. **分支预测策略**：预测不跳转（PC+4），猜错时冲刷流水线

## 2. GRE_array 流水线寄存器

### 2.1 模块设计

```verilog
module GRE_array #(
    parameter WIDTH = 200
)(
    input                   Clk,
    input                   Rst,
    input                   write_enable,
    input                   flush,
    input  [0:WIDTH-1]      in,
    output reg [0:WIDTH-1]  out
);
    always @(negedge Clk) begin
        if (write_enable) begin
            if (flush)
                out <= 0;
            else
                out <= in;
        end
    end

    always @(posedge Rst) begin
        out <= 0;
    end
endmodule
```

**要点**：
- **negedge Clk**：在时钟下降沿锁存 → 组合逻辑有半个周期稳定时间
- **write_enable**：`1'b1` = 正常流动，`1'b0` = 保持（插入气泡/NOP）
- **flush**：`1'b1` = 清零输出 → 等效插入 NOP 指令
- **Rst**：异步复位（posedge Rst），将所有寄存器输出清零

### 2.2 四个流水线段实例

```verilog
GRE_array #(.WIDTH(64))  IF_ID   (.Clk(clk), .Rst(rst),
    .write_enable(IF_ID_write_enable), .flush(IF_ID_flush),
    .in({...}), .out({...}));

GRE_array #(.WIDTH(200)) ID_EX   (.Clk(clk), .Rst(rst),
    .write_enable(1'b1), .flush(ID_EX_flush),
    .in({...}), .out({...}));

GRE_array #(.WIDTH(120)) EX_MEM  (.Clk(clk), .Rst(rst),
    .write_enable(1'b1), .flush(EX_MEM_flush),
    .in({...}), .out({...}));

GRE_array #(.WIDTH(110)) MEM_WB  (.Clk(clk), .Rst(rst),
    .write_enable(1'b1), .flush(1'b0),
    .in({...}), .out({...}));
```

**flush / write_enable 控制**：

| 场景 | IF_ID.write_enable | IF_ID.flush | ID_EX.flush | EX_MEM.flush |
|------|:---:|:---:|:---:|:---:|
| 正常流动 | 1 | 0 | 0 | 0 |
| Load-Use 阻塞 | **0** | 0 | **1** | 0 |
| 分支跳转 (EX 决议) | 1 | **1** | **1** | 0 |
| JAL (ID 决议) | 1 | **1** | 0 | 0 |

## 3. IF 级 — 取指

### 3.1 功能

- 从 ROM (IM) 取出指令
- 计算 PC + 4

### 3.2 数据通路

```
  NPC_next ──→ PC ──→ IM ──→ inst_in
                │
                └──→ +4 ──→ pcplus4
```

### 3.3 信号

| 信号 | 方向 | 位宽 | 说明 |
|------|------|------|------|
| `clk` | input | 1 | 时钟 |
| `rst` | input | 1 | 复位 |
| `NPC_next[31:0]` | input | 32 | 下一 PC（来自 NPC mux） |
| `PC[31:0]` | output | 32 | 当前 PC |
| `pcplus4[31:0]` | output | 32 | PC + 4 |
| `inst_in[31:0]` | input | 32 | 取出的指令（来自 ROM） |

### 3.4 NPC 选择逻辑

```
NPC_next = 
    EX_taken  ? EX_branch_target :   // EX 级分支跳转（最高优先级）
    ID_is_JAL ? ID_jal_target   :   // ID 级 JAL 跳转
                PC + 4;             // 默认：顺序执行
```

| 来源 | 信号 | 条件 | 优先级 |
|------|------|------|:---:|
| EX 级 | `EX_branch_target` | EX 级分支条件满足 或 JALR | 最高 |
| ID 级 | `ID_jal_target` | ID 级检测到 JAL | 中 |
| 默认 | `PC + 4` | 顺序执行 | 低 |

> `EX_branch_target` = 分支指令的 `PC + B-type 立即数`（从 ID/EX 中取出）  
> `ID_jal_target` = `PC + J-type 立即数`（EXT 在 ID 级计算）

### 3.5 代码模块映射

**PC.v**：保持现有，不做修改。输入 `NPC`（即上述 `NPC_next`），输出 `PC`。

## 4. ID 级 — 译码 / 读寄存器

### 4.1 功能

- 指令译码（ctrl）
- 读寄存器堆（RF）
- 立即数扩展（EXT）
- JAL 目标地址计算
- 冒险检测（hazard unit）

### 4.2 数据通路

```
IF_ID.instruction ──→ ctrl ──→ EXTOp, ALUOp, RegWrite, MemWrite, ...
                   │
                   ├──→ RF (A1=rs1, A2=rs2) ──→ RD1, RD2
                   │
                   └──→ EXT ──→ imm32
                   │
                   └──→ JAL_target = PC + J-immediate
```

### 4.3 指令字段提取（与单周期相同）

```verilog
assign rs1    = ID_instruction[19:15];
assign rs2    = ID_instruction[24:20];
assign rd     = ID_instruction[11:7];
assign Op     = ID_instruction[6:0];
assign Funct7 = ID_instruction[31:25];
assign Funct3 = ID_instruction[14:12];
```

### 4.4 冒险检测（Hazard Unit）

在 ID 级需要检测三种冒险：

#### (a) Load-Use 数据冒险（需阻塞）

```verilog
// 当 ID 级指令的 rs1 或 rs2 与 ID/EX 级指令的 rd 相同
// 且 ID/EX 级指令是 Load (WDSel == FromMEM)
wire load_use_hazard = (ID_EX.MemRead) &&
    ((ID_EX.rd != 0) && ((ID_EX.rd == rs1) || (ID_EX.rd == rs2)));
```

**处理**：阻塞 IF/ID，ID/EX 插入气泡（NOP）

#### (b) 普通数据冒险（转发解决）

```verilog
// EX 级转发：EX/MEM.rd == ID/EX.rs1 (且 EX/MEM.RegWrite)
// MEM 级转发：MEM/WB.rd == ID/EX.rs1 (且 MEM/WB.RegWrite)
```

#### (c) 控制冒险

- **JAL**：ID 级检测到 → IF_ID.flush = 1（刷掉 IF/ID 中刚取的指令），PC ← JAL_target
- **分支**：EX 级决议 → IF_ID.flush = 1，ID_EX.flush = 1（刷掉两条跟错分支的指令）

### 4.5 ctrl 模块

**完全复用现有 `ctrl.v`**（已支持全部 37 条指令），不做任何修改。

输入：`Op`, `Funct7`, `Funct3`, `Zero`（Zero 来自 EX 级，用于分支）
输出：`RegWrite`, `MemWrite`, `EXTOp[5:0]`, `ALUOp[4:0]`, `NPCOp[2:0]`, `ALUSrc`, `WDSel[1:0]`, `DMType[2:0]`

> **关键变化**：在流水线中，`NPCOp` 不再直接控制 NPC 模块，而是被 ID 级的 JAL 检测和 EX 级的分支决议替代。`NPCOp` 信号仅作为中间译码结果使用。

### 4.6 代码模块映射

| 原模块 | 改动 | 说明 |
|--------|------|------|
| `ctrl.v` | **不变** | 完全复用，输入输出接口不变 |
| `RF.v` | **微调** | `A3` 改为来自 `MEM_WB.rd`（写回地址经流水线延迟），`WD` 来自 WB 级 |
| `EXT.v` | **不变** | 完全复用 |

> RF 的关键变化：在流水线中，`RFWr` 和 `A3`、`WD` 来自 **MEM/WB 寄存器**（而非当前 ID 级指令），因为写回发生在 WB 级。

## 5. EX 级 — 执行

### 5.1 功能

- ALU 计算
- 前递多路选择（Forwarding MUX）
- 分支条件决议
- JALR 目标地址计算

### 5.2 数据通路

```
ID_EX.RD1 ──→ [ForwardA MUX] ──→ A ──┐
ID_EX.RD2 ──→ [ForwardB MUX] ──→ B ──┤
ID_EX.imm32 ─────────────────→ B ──┘  ├──→ ALU ──→ ALU_result
ID_EX.ALUOp ──────────────────────────┘         │
ID_EX.PC ─────────────────────────────→ AUIPC   │
                                                 │
                                   Zero ←───────┘
```

### 5.3 前递多路选择（Forwarding）

```verilog
// ForwardA: ALU A 操作数选择
wire [1:0] ForwardA;
// 00 = ID_EX.RD1 (无转发)
// 01 = EX_MEM.ALU_result (从 EX/MEM 转发)
// 10 = MEM_WB.WD (从 MEM/WB 转发)

assign ForwardA = (EX_MEM.RegWrite && EX_MEM.rd != 0 && EX_MEM.rd == ID_EX.rs1) ? 2'b01 :
                  (MEM_WB.RegWrite && MEM_WB.rd != 0 && MEM_WB.rd == ID_EX.rs1) ? 2'b10 : 2'b00;

// ForwardB: ALU B 操作数选择
wire [1:0] ForwardB;
// 00 = ID_EX.RD2 (无转发)
// 01 = EX_MEM.ALU_result (从 EX/MEM 转发)
// 10 = MEM_WB.WD (从 MEM/WB 转发)

assign ForwardB = (EX_MEM.RegWrite && EX_MEM.rd != 0 && EX_MEM.rd == ID_EX.rs2) ? 2'b01 :
                  (MEM_WB.RegWrite && MEM_WB.rd != 0 && MEM_WB.rd == ID_EX.rs2) ? 2'b10 : 2'b00;
```

**优先级**：EX/MEM > MEM/WB（最新结果优先，因为 EX/MEM 的指令更新）

```verilog
// ALU 输入（含转发 MUX）
wire [31:0] alu_A = (ForwardA == 2'b01) ? EX_MEM_ALU_result :
                    (ForwardA == 2'b10) ? WB_WD :
                                          ID_EX_RD1;

wire [31:0] alu_B_pre = (ID_EX_ALUSrc) ? ID_EX_imm32 : ID_EX_RD2;
wire [31:0] alu_B     = (ForwardB == 2'b01) ? EX_MEM_ALU_result :
                        (ForwardB == 2'b10) ? WB_WD :
                                              alu_B_pre;
```

### 5.4 分支决议

```verilog
// 分支条件检查（复用 ALU 比较结果）
// ALU 在 ALUOp=beq/bne/blt/... 时，C = {31'b0, condition}
wire branch_taken;
case (ID_EX_branch_type)  // 译码时记录的分支类型
    BEQ:  branch_taken = (alu_A == alu_B);
    BNE:  branch_taken = (alu_A != alu_B);
    BLT:  branch_taken = ($signed(alu_A) < $signed(alu_B));
    BGE:  branch_taken = ($signed(alu_A) >= $signed(alu_B));
    BLTU: branch_taken = (alu_A < alu_B);
    BGEU: branch_taken = (alu_A >= alu_B);
    JALR: branch_taken = 1'b1;
endcase

// EX 级跳转信号 → 送到 IF 级 NPC mux
assign EX_taken = (is_branch & branch_taken) | is_JALR;
assign EX_branch_target = is_JALR ? (alu_A + ID_EX_imm32) & ~32'b1  // JALR: rs1+imm, 清 LSB
                                  : ID_EX_branch_target;             // 分支: ID 级已算好
```

### 5.5 代码模块映射

| 原模块 | 改动 | 说明 |
|--------|------|------|
| `alu.v` | **微调** | 输入 `A`, `B` 现在来自转发 MUX（而非直连 RF）；接口不变，内部逻辑不变 |

## 7. WB 级 — 写回

### 7.1 功能

- 选择写回数据（ALU 结果 / MEM 数据 / PC+4）
- 写寄存器堆

### 7.2 写回数据选择（WD MUX）

```verilog
wire [31:0] WB_WD;
case (MEM_WB.WDSel)
    `WDSel_FromALU: WB_WD = MEM_WB.ALU_result;
    `WDSel_FromMEM: WB_WD = MEM_WB.DM_dout;
    `WDSel_FromPC:  WB_WD = MEM_WB.pcplus4;
    default:        WB_WD = 32'b0;
endcase
```

### 7.3 RF 写

```verilog
// RF 写使能和地址来自 MEM/WB 寄存器
RF U_RF (
    .clk(clk),
    .rst(reset),
    .RFWr(MEM_WB.RegWrite),
    .A1(rs1), .A2(rs2), .A3(MEM_WB.rd),     // A3 = 写回目标寄存器 (来自 WB 级!)
    .WD(WB_WD),                                // WD = 写回数据 (MUXX 后)
    .RD1(RD1), .RD2(RD2),
    .reg_sel(reg_sel),
    .reg_data(reg_data)
);
```

> **关键变化**：RF 的写地址 `A3` 和写数据 `WD` 来自 **MEM/WB 流水线寄存器**（滞后于 ID 级的读地址 `A1/A2`）。这是流水线的核心特性 —— 写回比译码晚 3 个周期。

## 9. 冒险处理

### 9.1 数据冒险：前递（Forwarding）

#### 前递路径

```
  EX/MEM.ALU_result ─────────────────────────┐
                                              ├──→ ForwardA/B MUX (EX 级)
  MEM/WB.WD (MUX 后) ────────────────────────┘
```

#### 前递条件（以 ForwardA 为例）

```verilog
// EX 危险：上一条指令 (EX/MEM) 的目标寄存器 == 当前指令 (ID/EX) 的 rs1
wire ex_hazard_rs1  = EX_MEM.RegWrite && (EX_MEM.rd != 0) && (EX_MEM.rd == ID_EX.rs1);

// MEM 危险：上上条指令 (MEM/WB) 的目标寄存器 == 当前指令的 rs1
wire mem_hazard_rs1 = MEM_WB.RegWrite && (MEM_WB.rd != 0) && (MEM_WB.rd == ID_EX.rs1);

// 优先级：EX > MEM (更近的指令数据更新)
assign ForwardA = ex_hazard_rs1  ? 2'b01 :   // 转发 EX/MEM.ALU_result
                  mem_hazard_rs1 ? 2'b10 :   // 转发 WB_WD
                                   2'b00;    // 不转发 (用 ID_EX.RD1)
```

### 9.2 数据冒险：Load-Use 阻塞

当 Load 指令后紧跟使用其结果的指令时，转发无法解决（数据在 MEM 级才就绪，但下一条指令在 EX 级就需要）：

```
lw   x1, 0(x2)     ← 数据在 MEM 级才从 DM 读出
addi x3, x1, 5     ← 在 EX 级就需要 x1 → 只能阻塞！
```

#### 检测条件

```verilog
wire load_use_hazard = ID_EX.MemRead &&                          // ID/EX 是 Load
    ((ID_EX.rd == rs1 && rs1 != 0) || (ID_EX.rd == rs2 && rs2 != 0));  // 且与 ID 级 rs1/rs2 相同
```

#### 阻塞动作（1 cycle）

| 信号 | 值 | 效果 |
|------|:---:|------|
| IF_ID.write_enable | 0 | IF/ID 保持不变（PC 也需保持不变） |
| ID_EX.flush | 1 | ID/EX 变为 NOP（气泡） |

```
周期1:  lw   x1,0(x2)  IF    ID    EX    MEM   WB
周期2:  addi x3,x1,5   IF    ID    --    --    --
         ↑ 此处检测到 load_use_hazard
周期3:  addi x3,x1,5   IF    ID    EX    MEM   WB   ← 转发 MEM→EX 解决！
周期4:  (下一条)        IF    ID    EX    ...
```

### 9.3 控制冒险：分支预测

**策略：预测不跳转**（最简单的静态预测）

#### JAL（ID 级检测）

```
JAL 在 ID 级译码 → 立即计算目标 → 冲刷 IF/ID
```

```verilog
wire ID_is_JAL = (ID_instruction[6:0] == 7'b1101111);  // opcode=JAL
assign ID_jal_target = ID_pcplus4 - 32'd4 + ID_jal_imm; // PC + J-immediate

// 控制信号
assign IF_ID_flush = ID_is_JAL;         // 冲刷 IF/ID 中已取的下一条错误指令
// PC 在下个 posedge 取 ID_jal_target
```

> 代价：1 cycle（IF/ID 中的错误指令被 flush 掉）

#### 分支指令（BEQ/BNE/BLT/BGE/BLTU/BGEU，EX 级决议）

```
分支在 EX 级计算出条件 → 若不跳转正常流动 → 若跳转冲刷 IF/ID 和 ID/EX
```

```verilog
// EX 级
wire EX_is_branch = (ID_EX.branch_type != 3'b000) && (ID_EX.branch_type != 3'b111);
wire EX_is_JALR   = (ID_EX.branch_type == 3'b111);
wire EX_taken = EX_is_JALR | (EX_is_branch && branch_condition_met);

// 跳转时冲刷
assign IF_ID_flush  = EX_taken;    // 冲刷 IF/ID
assign ID_EX_flush  = EX_taken;    // 冲刷 ID/EX
```

> 代价：2 cycles（IF/ID 和 ID/EX 各一条错误指令被 flush 掉）

#### 冲刷汇总逻辑

```verilog
// 最终冲刷信号（多源 OR）
wire IF_ID_flush  = ID_is_JAL | EX_taken;
wire ID_EX_flush  = load_use_hazard | EX_taken;
```
