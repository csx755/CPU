## 1. 项目概况

### 1.1 最终目标

在 A7 FPGA 开发板上实现一个 **5 级流水线 RISC-V 处理器**，完整支持 **RV32I 基指令集**（共 37 条指令）。

### 1.2 分阶段路线

```
┌─────────────────────────────────────────────────────────────┐
│ 阶段一   8指令单周期    →  仿真验证通过  →  下板验证通过    │
├─────────────────────────────────────────────────────────────┤
│ 阶段二   扩展至37指令   →  仿真验证通过  →  下板验证通过    │
│          (RV32I 完整)                                       │
├─────────────────────────────────────────────────────────────┤
│ 阶段三   5级流水线实现  →  仿真验证通过  →  下板验证通过    │
│          (IF/ID/EX/MEM/WB + 冒险处理)                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 开发工具

| 工具 | 用途 | 说明 |
|------|------|------|
| **Vivado** | 综合/实现/下板 | FPGA 厂商工具，含仿真器 |
| **VS Code** | 代码编辑 | 配合 Verilog 插件 |
| **Git** | 版本控制 | 分阶段标记，支持回溯 |
| **ModelSim** (备选) | 仿真 | 可配合 Vivado 使用 |

### 2.3 架构说明（单周期）

当前 CPU 是**单周期微架构**——每条指令在一个时钟周期内完成全部 5 个阶段：

```
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│ IF   │→ │ ID   │→ │ EX   │→ │ MEM  │→ │ WB   │
│取指   │  │译码   │  │执行   │  │访存   │  │写回   │
└──────┘  └──────┘  └──────┘  └──────┘  └──────┘
←────────── 一个时钟周期 ──────────→
```

特点是控制逻辑简单，但时钟频率受限于最慢指令（通常是 LW，因需取指+译码+ALU+访存+写回）。

### 2.4 关键数据通路

```
PC → IM(取指) → ctrl(解码)
              → RF(读寄存器) + EXT(扩立即数)
              → ALU(计算)
              → DM(load/store) 或 直接写回RF
NPC 计算下一地址
```

## 6. 阶段三：实现流水线 CPU

### 6.1 架构设计

将单周期拆分为 5 级流水线：

```
            IF         ID          EX          MEM         WB
       ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
       │ 取指    │ │ 译码/读 │ │ 执行    │ │ 访存    │ │ 写回    │
       │         │ │ 寄存器  │ │         │ │         │ │         │
       │ PC→IM  │ │ Ctrl    │ │ ALU     │ │ DM      │ │ RF[rd]←│
       │        │ │ RF读    │ │         │ │         │ │         │
       │ PC+4   │ │ EXT     │ │ NPC     │ │         │ │         │
       └────────┘ └────────┘ └────────┘ └────────┘ └────────┘
           │          │          │          │          │
       ┌───▼───┐ ┌───▼───┐ ┌───▼───┐ ┌───▼───┐
       │IF/ID  │ │ID/EX  │ │EX/MEM │ │MEM/WB │
       │寄存器  │ │寄存器  │ │寄存器  │ │寄存器  │
       └───────┘ └───────┘ └───────┘ └───────┘
```

### 6.2 关键设计要点

#### 流水线寄存器

| 寄存器 | 存储内容 |
|--------|---------|
| IF/ID | PC+4，指令码，PC 值 |
| ID/EX | 控制信号，rs1/rs2/rd 地址，RD1/RD2，立即数，PC+4 |
| EX/MEM | 控制信号，ALU 结果，RD2，rd 地址 |
| MEM/WB | 控制信号，DM 读出数据，ALU 结果，rd 地址 |

#### 冒险处理

**数据冒险**（RAW 依赖）：
- **方法一：前递（Forwarding）** —— 从 EX/MEM 或 MEM/WB 直接将结果送到 ALU 输入
- **方法二：插入气泡（Stall）** —— 当 LW 后紧接使用结果时（Load-Use）需要暂停一周期

**控制冒险**（分支跳转）：
- **分支预测** —— 简单策略：预测不跳转（继续 PC+4）
- 跳转时刷新 IF/ID 流水线寄存器

### 6.3 实现步骤

```
第 1 步：代码结构重构
  sccomp_pipeline.v (新顶层)
  └── IF_stage.v          取指级 (PC + IM)
  └── IF_ID_reg.v         IF/ID 流水线寄存器
  └── ID_stage.v          译码级 (ctrl + RF + EXT)
  └── ID_EX_reg.v         ID/EX 流水线寄存器
  └── EX_stage.v          执行级 (ALU + NPC)
  └── EX_MEM_reg.v        EX/MEM 流水线寄存器
  └── MEM_stage.v         访存级 (DM)
  └── MEM_WB_reg.v        MEM/WB 流水线寄存器
  └── WB_stage.v          写回级
  └── hazard_unit.v       冒险检测与转发单元

第 2 步：逐级添加流水线寄存器
  从 IF/ID 开始，每完成一级就跑仿真验证

第 3 步：数据前递
  ForwardA/ForwardB 解决 EX 和 MEM 的 RAW 冒险

第 4 步：Load-Use 冒险
  LW 后紧跟使用 → 插入一个气泡周期

第 5 步：分支冒险
  简化：分支结果在 EX 级确定，预测不跳转
  误预测 → 刷掉 IF/ID 中的指令

第 6 步：综合优化
  调整时序，确保流水线路径无关键路径违例
```

### 6.4 验证策略

- **IPC 验证**：运行同一测试程序，流水线版应比单周期版有更少的周期数
- **正确性验证**：寄存器堆最终状态与单周期版本一致
- **冒险测试**：专门构造数据冒险和控制冒险的汇编测试
- **仿真波形**：逐级观察流水线寄存器中的指令流动

### 6.5 对照验证程序（汇编建议）

```
# 数据冒险测试（RAW）
addi x1, x0, 10
addi x2, x1, 5    # 依赖 x1
sub  x3, x2, x1   # 依赖 x2 和 x1

# Load-Use 测试
lw   x4, 0(x0)
addi x5, x4, 1    # 依赖 x4 → 前递

# 分支预测测试
addi x1, x0, 5
beq  x1, x0, _skip
addi x2, x0, 10   # 误预测会被冲刷
_skip:
```

## 附录 A：RISC-V 指令格式速查

### R-type（寄存器-寄存器）

```
┌─────┬─────┬─────┬─────┬─────┬─────────┐
│funct7│ rs2 │ rs1 │funct3│ rd  │ opcode  │
│ 7bit │ 5bit│ 5bit│ 3bit │ 5bit│  7bit   │
│[31:25]│24:20│19:15│14:12│11:7 │  6:0    │
└─────┴─────┴─────┴─────┴─────┴─────────┘
```

### I-type（立即数/加载）

```
┌──────────┬─────┬─────┬─────┬─────────┐
│ imm[11:0]│ rs1 │funct3│ rd  │ opcode  │
│  12bit   │ 5bit│ 3bit │ 5bit│  7bit   │
└──────────┴─────┴─────┴─────┴─────────┘
```

### S-type（存储）

```
┌──────┬─────┬─────┬─────┬──────┬─────────┐
```
